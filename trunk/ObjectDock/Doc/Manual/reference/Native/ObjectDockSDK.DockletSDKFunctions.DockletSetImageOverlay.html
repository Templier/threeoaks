DockletSetImageOverlay

Set the image overlay.

void DockletSetImageOverlay(
   HWND hwndDocklet,
   Image *imageOverlay,
   BOOL bAutomaticallyDeleteImage = TRUE
);

Parameters

hwndDocklet
The handle to the window passed during OnCreate that owns this docklet and therefore identifies the docklet to the host.
imageOverlay
[in] Specifies the new overlay image for the docklet, which will draw directly and seamlessly over any image set using DockletSetImage or DockletSetImageFile.
bAutomaticallyDeleteImage
Upon the next call to DockletSetImage, DockletSetImageFile, etc, ObjectDock would by default delete the image set to it by this function. Set this flag to FALSE to prevent ObjectDock from deleting this image, thus allowing this image to be reused after it is no longer being displayed. By default, set to TRUE - yes delete the image after no longer in use.

Remarks

In ObjectDock, instead of dealing with Paint messages, ObjectDock allows you to set an image for your Docklet at any time, as well as setting an 'overlay' image over to top of this base image. This allows for such things as a base background image to be constantly loaded, and then for details to be added on top of this image, for example a CPU meter on top of a holder, or a mail notification icon with a number on top of a mail icon loaded from a file, which will not have to be reloaded when mail arrives and the overlay image needs to be changed.
