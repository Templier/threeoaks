ObjectDock SDK Reference

OnCreate

A mandatory function, this function is called every time a docklet from this dll is created. Provides the handle of the window the docklet will be represented by, along with the instance of the docklet's dll. If the plugin is being loaded for the first time, szIni and szIniGroup will be NULL, however otherwise they will specify the ini file and ini section the docklet should load its individual options from.

Since multiple of the same ObjectDock docklets can be added at the same time, to allow each to have their own properties, a structure or class pointer should be returned from OnCreate to keep track of this individual plugin's properties.

DOCKLET_DATA *CALLBACK OnCreate(
   HWND hwndDocklet,
   HINSTANCE hInstance,
   char *szIni,
   char *szIniGroup
);

Parameters

hwndDocklet
The handle to the window that owns this docklet and therefore identifies the docklet to the host.
This handle should be stored as it will be needed to manipulate the docklet when communicating with the host.
hInstance
Handle to the instance of this docklet DLL.
szIni
Absolute path of an INI file which the plugin should use to load options from.
Can be NULL. If NULL, then the plugin is a new plugin and default options should be loaded.
szIniGroup
The INI Section of an INI file which the plugin should use to load options from.
Can be NULL. If NULL, then the plugin is a new plugin and default options should be loaded.

Return Value

You should return a pointer to a location in memory which this plugin will use to store it's personal "per-plugin" settings.
The pointer returned will be passed with every further call to the plugin as to identify the specific instance of that plugin.
(Returned datatype is arbitrary)

Useful examples are returning a pointer to a structure or a class which can store information about this individual plugin, information such as owning hwnd and state data.
